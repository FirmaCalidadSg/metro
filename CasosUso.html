Casos de Uso - SICATEAM

1. Gestión de Usuarios y Accesos

CU-01: Iniciar Sesión

Actor Principal: Usuario del Sistema
Precondiciones: Usuario registrado en el sistema
Flujo Principal:
  1. Usuario accede a la página de inicio
  2. Sistema muestra formulario de login
  3. Usuario ingresa credenciales
  4. Sistema valida credenciales
  5. Sistema permite acceso y muestra menú principal
Flujos Alternativos:
  • Credenciales inválidas: Sistema muestra mensaje de error
  • Cuenta bloqueada: Sistema notifica al usuario

CU-02: Gestionar Perfiles

Actor Principal: Administrador
Precondiciones: Usuario autenticado con permisos de administrador
Flujo Principal:
  1. Administrador accede a "Profile" en menú Administración
  2. Sistema muestra lista de perfiles
  3. Administrador puede crear/modificar/eliminar perfiles
  4. Sistema actualiza cambios en tiempo real
Flujos Alternativos:
  • Perfil en uso: Sistema impide eliminación

2. Gestión de Plantas

CU-03: Registrar Nueva Planta

Actor Principal: Supervisor de Plantas
Precondiciones: Usuario autenticado con permisos de supervisor
Flujo Principal:
  1. Supervisor selecciona "Planta" en menú Administración
  2. Sistema muestra formulario de registro
  3. Supervisor ingresa datos de la planta:
    • Nombre y ubicación
    • Capacidad instalada
    • Equipos asociados
  4. Sistema valida y guarda información
Flujos Alternativos:
  • Datos incompletos: Sistema solicita información faltante
  • Planta duplicada: Sistema notifica error

CU-04: Monitorear Capacidad de Planta

Actor Principal: Supervisor de Planta
Precondiciones: Planta registrada en el sistema
Flujo Principal:
  1. Supervisor accede a "Dashboard de Capacidad"
  2. Sistema muestra métricas en tiempo real:
    • Capacidad utilizada
    • Eficiencia por línea
    • Tiempos muertos
  3. Supervisor puede filtrar por períodos
  4. Sistema actualiza visualizaciones
Flujos Alternativos:
  • Sin datos: Sistema muestra mensaje informativo
  • Error de conexión: Sistema intenta reconectar

CU-05: Registrar Pérdidas

Actor Principal: Operador
Precondiciones: Usuario autenticado con acceso a registro
Flujo Principal:
  1. Operador selecciona "Registro de Pérdidas"
  2. Sistema muestra formulario de registro
  3. Operador ingresa detalles:
    • Tipo de pérdida
    • Duración
    • Causa raíz
  4. Sistema almacena información
Flujos Alternativos:
  • Datos inválidos: Sistema solicita corrección

CU-06: Análisis de Producción

Actor Principal: Supervisor de Producción
Precondiciones: Datos de producción disponibles
Flujo Principal:
  1. Supervisor accede a "Análisis de Producción"
  2. Sistema muestra panel con:
    • Producción por turno
    • Cumplimiento de objetivos
    • Tendencias de producción
  3. Supervisor selecciona período de análisis
  4. Sistema genera reportes comparativos
Flujos Alternativos:
  • Sin datos en período: Sistema muestra mensaje
  • Error en cálculos: Sistema notifica inconsistencias

CU-07: Monitoreo de Tiempos Perdidos

Actor Principal: Supervisor de Planta
Precondiciones: Sistema en funcionamiento
Flujo Principal:
  1. Supervisor accede a "Monitor de Tiempos"
  2. Sistema presenta:
    • Tiempo inactivo por equipo
    • Estadísticas por turno
    • Patrones identificados
  3. Sistema genera recomendaciones automáticas
Flujos Alternativos:
  • Fallo de sensor: Sistema marca datos como estimados

CU-08: Gestión de Alertas

Actor Principal: Operador de Sistema
Precondiciones: Sistema de alertas configurado
Flujo Principal:
  1. Sistema detecta condición de alerta
  2. Sistema notifica:
    • Tipo de alerta
    • Ubicación/Equipo
    • Nivel de criticidad
  3. Operador confirma recepción
  4. Sistema registra respuesta
Flujos Alternativos:
  • Alerta falsa: Operador marca como falso positivo

CU-09: Generación de Reportes

Actor Principal: Usuario Autorizado
Precondiciones: Acceso a módulo de reportes
Flujo Principal:
  1. Usuario selecciona tipo de reporte
  2. Sistema muestra opciones:
    • Período de tiempo
    • Formato de salida
    • Nivel de detalle
  3. Sistema genera documento
  4. Usuario descarga reporte
Flujos Alternativos:
  • Error de generación: Sistema ofrece retry

CU-10: Mantenimiento de Catálogos

Actor Principal: Administrador
Precondiciones: Acceso administrativo
Flujo Principal:
  1. Administrador accede a catálogos
  2. Sistema muestra opciones:
    • Equipos
    • Tipos de fallas
    • Categorías
  3. Administrador realiza mantenimiento
  4. Sistema actualiza registros
Flujos Alternativos:
  • Registro duplicado: Sistema notifica conflicto

CU-11: Gestión de Usuarios

Actor Principal: Administrador del Sistema
Precondiciones: Acceso total al sistema
Flujo Principal:
  1. Administrador accede a gestión de usuarios
  2. Sistema muestra opciones:
    • Alta/Baja usuarios
    • Asignación de roles
    • Permisos especiales
  3. Administrador modifica configuración
  4. Sistema aplica cambios
Flujos Alternativos:
  • Usuario existente: Sistema solicita confirmación
  • Error en permisos: Sistema mantiene configuración previa

CU-12: Configurar Parámetros del Sistema

Actor Principal: Administrador del Sistema
Precondiciones: Acceso administrativo
Flujo Principal:
  1. Administrador accede a configuraciones
  2. Sistema muestra parámetros:
    • Intervalos de backup
    • Umbrales de alertas
    • Configuración de correos
  3. Sistema aplica cambios en tiempo real
Flujos Alternativos:
  • Error de configuración: Sistema mantiene valores anteriores
